# ✂ LinkCutter

> **Краткое описание:** ✂️🔗 Быстрый и современный сервис для сокращения URL-адресов. Наподобие Bitly. Проект построен на базе асинхронного стека Python, обеспечивая высокую производительность и легкость развертывания.

## 🛠 Техно-стек

- **Язык:** `Python 3.13`
- **Web-фреймворк:** `FastAPI` + `uvicorn`
- **База данных:** `PostgreSQL` (`SQLAlchemy` + `asyncpg`)
- **Валидация данных:** `Pydantic`
- **Пакетный менеджер:** `uv` (`hatchling` под капотом)
- **Инфраструктура:** `Docker` & `Docker Compose`
- **Нагрузочное тестирование:** `Locust`

## 🛠 Запуск проекта

### 🐳 Запуск через Docker (Рекомендуется)

Убедитесь, что у вас установлены Docker и Docker Compose. Проект настроен на автоматическую сборку без необходимости
локальной установки Python-окружения.

---

! Важное уточнение. Если вы добавили в проект новую зависимость. Перед сборкой рекомендуется выполнить(поскольку в
контейнере проект запускается без uv):

```shell
uv pip compile pyproject.toml -o requirements.txt
```

---

Продолжаем...

1. Клонируйте репозиторий:

   ```shell
   git clone "https://github.com/Sh1yden/LinkCutter.git"
   cd LinkCutter
   ```

2. Запустите докер:

Если используете `docker` в `windows`, запустите приложение. Если используете в `linux`, используйте команду ниже чтобы
проверить скачан ли он у вас. Если нет, скачайте и продолжите с `3` пункта.

```shell
docker --version
```

3. Создайте docker контейнер с приложением:

   ```shell
   docker-compose up -d
   ```

4. Зайдите на `localhost:8000` или `127.0.0.1:8000` в своём браузере.
5. Пользуйтесь 💜.

### 💻 Локальный запуск

#### 1. Клонируйте репозиторий:

```shell
git clone "https://github.com/Sh1yden/LinkCutter.git"
cd LinkCutter
```

#### 2. Разбираемся с uv:

##### Инициализируйте виртуальное окружение:

```shell
uv init
```

##### Закрепление версии:

```shell
uv python pin 313
```

##### Создание виртуального окружения:

```shell
uv venv .venv
```

##### Синхронизируйте его:

```shell
uv sync
```

#### 3. Запуск

Локальный запуск через `uvicorn`.
Флаг `--reload` отвечает за постоянную перезагрузку `API` при изменении кода.

```shell
uvicorn app.main:app --reload
```

## 🗄 Структура проекта

- `app` - папка всего приложения, или же `API`.
  - `core` - ядро проекта. тут расположен `logger` и конфиги(настройки) приложения.
  - `database` - вся работа с бд.
    - `core` - ядро базы. её инициализация находиться тут.
    - `database files` - папка на случай если проект будет использовать `sqlite`.
    - `migrations` - отдельная папка для `alembic` миграций.
    - `models` - все модели бд.
  - `schemas` - `pydantic` схемы для `API` и проверок в коде.
  - `services`
    - вся бизнес логика, не закапсулирована в виде repo патерна, а просто в виде функций в файле `url_service.py`,
      поскольку проект большой и это уже получается оверинжиниринг.
    - `base62` - способ сокращения ссылки.
  - `utils` - папка с возможными утилами для проекта.

  - `main.py` - основной файлик запуска приложения.

- `locustfiles` - файлы для нагрузочного тестирования.
  - [README.md](locustfiles/README.md) - описание нагрузочного тестирования.
- `tests` - тесты проекта.
- `pyptoject.toml` - конфигурация проекта.
- `Dockerfile` и `docker-compose` инфраструктурные файлы для деплоя.
- [`README.md`](README.md) - этот файл.
- [`CHANGELOG.md`](CHANGELOG.md) - список изменений в проекте.
- [`LICENSE`](LICENSE) - лицензия проекта.
