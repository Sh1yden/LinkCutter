# LinkCutter ✂️🔗

Быстрый и современный сервис для сокращения URL-адресов. Наподобие [Bitly](https://bitly.com/). Проект построен на базе асинхронного стека Python, обеспечивая высокую производительность и легкость развертывания.

## ❔ Зачем вообще проект?

Это [тестовое задание](https://github.com/avito-tech/auto-backend-trainee-assignment) на позицию стажера/junior backend от авито.

## 🚀 Технологии

* **Язык:** `Python 3.13`
* **Web-фреймворк:** `FastAPI`
* **База данных:** `PostgreSQL` (`SQLAlchemy` + `asyncpg`)
* **Валидация данных:** `Pydantic`
* **Пакетный менеджер:** [uv](https://github.com/astral-sh/uv) (и `hatchling` под капотом)
* **Инфраструктура:** `Docker` & `Docker Compose`

## 🛠 Запуск проекта

### 🐳 Запуск через Docker (Рекомендуется)

Убедитесь, что у вас установлены Docker и Docker Compose. Проект настроен на автоматическую сборку без необходимости локальной установки Python-окружения.

1. Клонируйте репозиторий:
   
   ```shell
   git clone "https://github.com/Sh1yden/LinkCutter.git"
   cd LinkCutter
   ```

2. Запустите докер:

Если используете `docker` в `windows`, запустите приложение. Если используете в `linux`, используйте команду ниже чтобы проверить скачан ли он у вас. Если нет, скачайте и продолжите с `3` пункта.

```shell
docker --version
```

3. Создайте docker контейнер с приложением:

   ```shell
   docker-compose up -d
   ```
   
4. Зайдите на `localhost:8000` в своём браузере.
5. Пользуйтесь 💜.

### 🖥 Локальный запуск

#### 1. Клонируйте репозиторий:

   ```shell
   git clone "https://github.com/Sh1yden/LinkCutter.git"
   cd LinkCutter
   ```
   
#### 2. Разбираемся с uv:

##### Инициализируйте виртуальное окружение:
```shell
uv init
```

##### Закрепление версии:

```shell
uv python pin 313
```

##### Создание виртуального окружения:

```shell
uv venv .venv
```

##### Синхронизируйте его:

```shell
uv sync
```

#### 3. Запуск

Локальный запуск через `uvicorn`.
Флаг `--reload` отвечает за постоянную перезагрузку [[API]] при изменении кода. 

```shell
uvicorn app.main:app --reload
```

## 🗄 Структура проекта
- `app` - папка всего приложения, или же [[API]].
	- `core` - ядро проекта. тут расположен `logger` и конфиги(настройки) приложения.
	- `database` - вся работа с бд.
		- `core` - ядро базы. её инициализация находиться тут.
		- `database files` - папка на случай если проект будет использовать `sqlite`.
		- `migrations` - отдельная папка для `alembic` миграций.
		- `models` - все модели бд.
		- `schemas` - `pydantic` схемы для [[API]] и проверок в коде.
	- `services` 
		- вся бизнес логика, не закапсулирована в виде repo патерна, а просто в виде функций в файле `url_service.py`, поскольку проект большой и это уже получается оверинжиниринг.
		- `base62` - способ сокращения ссылки.
	- `utils` - папка с возможными утилами для проекта.
	- `main.py` - основной файлик запуска приложения.
- `pyptoject.toml` - конфигурация проекта.
- `Dockerfile` и `docker-compose` инфраструктурные файлы для деплоя.
